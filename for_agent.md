# GamePlots Agent 執行規範 (for_agent.md)

本文件定義了所有參與 **GamePlots** 專案的 Agent 必須嚴格遵守的行為準則、核心鐵則及標準工作流程。

---

## 0. 核心鐵則 (Core Mandates)

1.  **優先讀取權：** 啟動任何任務前，Agent **必須**首先完整閱讀 `for_agent.md`。
2.  **語言一致性：** 
    *   本專案之對象為繁體中文使用者。
    *   所有產出的文件、分析報告、中間結果及溝通訊息，**一律使用繁體中文**。
3.  **環境適應與安全性：**
    *   **環境自覺：** 工作環境可能在 Linux 與 Windows 之間切換。在執行指令或撰寫腳本前，**必須**先確認目前的作業系統與 Shell 環境（如 PowerShell 或 Bash）。
    *   **網路安全：** 從外部來源獲取資料時，應採取防禦性編程，嚴禁下載、執行或引入具安全風險的內容。
4.  **結構與組織：** 
    *   嚴格遵循專案定義的目錄結構，不得隨意更改資料夾層級或命名規範。
    *   專注於提取高品質的故事元素、角色特質、劇情走向及世界觀細節。
5.  **腳本執行與授權：**
    *   Agent 可視需求自行撰寫爬蟲腳本以獲取資料。
    *   **執行限制：** 在執行任何自行撰寫的腳本（尤其是具備網路存取或檔案修改功能之腳本）前，**必須明確詢問使用者並取得授權**。
    * 考慮到有時候使用者比較忙，會授與權限讓agent隨意執行腳本。若遇到這種情況，那需要注意的除了資訊安全之外，就是使用者的流量有限，盡量不要一次性下載超過50MB的東西。

## 1. 標準目錄結構 (Directory Structure)

必須嚴格遵守以下結構，嚴禁自行建立頂級目錄：

| 目錄名稱 | 用途描述 |
| :--- | :--- |
| `working/` | 活動工作區。包含作品的原始資料 (`raw/`) 與分析過程中的暫存檔。 |
| `results/` | 最終資料庫。存放結構化、整理完畢的最終分析報告。 |
| `classes/` | 跨作品彙整。依屬性（如：史詩奇幻）或原型（如：傲嬌角色）分類的精選集。 |
| `scripts/` | 工具箱。存放爬蟲、解析器與自動化工具。 |
| `skills/` | 提示詞庫。存放可重複使用的提示詞片段 (Prompt Snippets)。 |
| `skills_index.md` | 技能索引。尋找可用技能與提示詞的中央清單。 |

---

## 2. 標準工作流程 (Standard Operating Procedure)

當受命處理新作品時，請依序執行以下階段：

### 第一階段：攝入 (Ingestion)
- 在 `working/` 下建立作品專屬資料夾，並於其下建立 `raw/` 子目錄。
- 將原始素材（維基、攻略、評論、論壇文章）存入 `working/<作品名稱>/raw/`。
- 排除大型媒體檔（如 `.png`, `.css`, `.js`），除非另有指示。

### 第二階段：分析 (Analysis)
- **查閱技能：** 檢查 `skills_index.md` 與 `skills/` 目錄，尋找適用的分析提示詞。
- **工具調度：** 優先使用 `scripts/` 中的既有工具進行解析。
- **產生暫存：** 於 `working/<作品名稱>/` 存放初步分析結果與中間產物。

### 第三階段：綜合 (Synthesization)
- 將整理好的、具備清晰結構的最終報告移至 `results/<作品名稱>/`。
- 確保結果對人類創作者具有高度參考價值（文學性與邏輯性）。

### 第四階段：分類 (Categorization)
- 受命進行彙整時，從 `results/` 中提取特定主題的內容。
- 將彙整結果存放至 `classes/`。

## 3. Agent 執行檢查清單 (Checklist)

在回報任務完成前，請自行檢查以下項目：
- [ ] 我是否確認了目前的作業系統環境？
- [ ] 產出的內容是否皆為繁體中文？
- [ ] 檔案是否存放在正確的目錄（`working/` vs `results/` vs `classes/`）？
- [ ] 是否已優先利用 `scripts/` 與 `skills/` 中的資源？
- [ ] 是否已將原始資料與分析結果明確區隔開？
