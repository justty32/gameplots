我對於各類遊戲的背景故事、人物故事、劇情等，包括介紹之類，都很有興趣。
所以這個專案就是用來收集這些東西的。我會讓agents來負責。
比如我會指定某個遊戲，然後請agent去網路上下載相關的wiki、攻略、心得、資料之類
然後讓agent依據這些進行分析，提取出有用的東西。

也不只遊戲，漫畫、小說、電影、動畫等都會有。

我是打算拿這些來做參考，用於設計我自己的遊戲的背景、人物、劇情之類。

有幾個要點：

  這個專案底下會有多個遊戲的分析。所以這個專案底下會有幾個資料夾：
    working/ 這個資料夾底下，各作品名稱都會有自己的子資料夾，那就是該作品的各種原始資料(從網路上抓來的或我給的)的存放地方，還有分析的暫時結果的存放地方。
    > 各作品的子資料夾內，又會有raw/，這個就是用來存放原始資料的地方。其他就是存放暫時分析結果的地方
    > 一些檔案會被放在.gitignore中，比如.png之類占空間的，或是太大的.css, javascript之類
    scripts/ 這個資料夾底下，是各種可使用的工具如爬蟲、解析器
    skills/ 這裏面放著一些好用的提示詞片段
    skills_index.md，放著提示詞片段的index，方便agent取用
    results/ 這個資料夾底下存放著組織好的分析與整理結果。各作品名稱都會有自己的子資料夾。
    classes/ 這裡放著一些依據奇幻背景、科幻背景、二次元角色之類的分類，從results中整理出的資料集合。比如我會說請agent幫我從results中挖出一些奇幻背景的作品的史詩故事，集合弄成一個.md放到這裡。或是挖出一些二次元性格的角色的介紹，集合成一個.md放到這裡。
    for_agent.md 我會讓所有agent在開始動工前，都先讀取這個檔案。裡面放著一些規則、鐵則、工作流規定之類

  為了讓agent能夠更有效的從網路上抓東西，我可能會手動提供一些網址，然後讓agent寫爬蟲去抓
  或是提供一些關鍵詞，讓agent自己去搜尋，然後下載。
  > 要非常注意安全
