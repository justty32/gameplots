我對於各類遊戲的背景故事、人物故事、劇情等（包括介紹之類）都很有興趣。
因此這個專案是用來收集這些內容的，並由 agents 負責執行。
例如，我會指定某個遊戲，然後請 agent 從網路上取得相關的 wiki、攻略、心得、資料等，
接著讓 agent 根據這些內容進行分析，提取出有用的資訊。

不僅限於遊戲，漫畫、小說、電影、動畫等也都會包含在內。

我的目的是將這些資料作為參考，用於設計我個人遊戲的背景、人物與劇情。

### 專案要點：

- **目錄結構：** 本專案會包含多個作品的分析，結構如下：
    - `working/`：各作品的專屬資料夾。存放該作品的原始資料（從網路抓取或由我提供）以及分析的暫時結果。
        - 每個作品子資料夾內包含 `raw/`：專門存放原始資料。其他空間則存放暫時分析結果。
        - 部分檔案會被加入 `.gitignore`，例如佔空間的 `.png`，或過大的 `.css`、`.javascript` 等。
    - `scripts/`：存放各種可用工具，如爬蟲、解析器等。
    - `skills/`：存放實用的提示詞（prompt）片段。
    - `skills_index.md`：提示詞片段的索引，方便 agent 存取。
    - `results/`：存放整理好的分析與彙整結果。每個作品有其專屬資料夾。
    - `classes/`：存放依據背景（如奇幻、科幻）或角色類型（如二次元角色）分類的彙整資料。例如：請 agent 從 `results/` 中挖出奇幻背景作品的史詩故事並彙整至此，或彙整二次元性格角色的介紹。
    - `for_agent.md`：所有 agent 在動工前必須讀取的檔案。內含規則、鐵則與工作流規定。

- **資料獲取：** 為了讓 agent 更有效率地從網路抓取資料，我可能會提供特定網址讓 agent 撰寫爬蟲，或提供關鍵字讓 agent 自行搜尋並下載。
- **安全性：** 執行下載與抓取任務時必須極度注意安全。
