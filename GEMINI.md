# GamePlots 專案操作指南 (GEMINI.md)

本文件為 **GamePlots** 專案的核心指令集與架構指南。所有參與本專案的 Agent 與協作者均須遵守此處定義的規範，以確保敘事資料分析的一致性與高品質產出。

---

## 0. 專案願景與目標

**GamePlots** 是一個專門用於收集、處理並分析各類媒介（遊戲、漫畫、小說、影視）敘事資料的協作環境。
其核心目標是建立一個結構化的故事元素庫（背景、角色、劇情），為原創遊戲設計提供深度的靈感參考。

---

## 1. 核心開發與執行規範

*   **語言一致性：** 專案使用者為繁體中文使用者。所有文件、分析報告、中間產物及通訊訊息**必須使用繁體中文**。
*   **Agent 初始化：** 在執行任何具體任務前，Agent **必須**優先閱讀並理解 `for_agent.md` 中的「執行規範」與「核心鐵則」。
*   **環境警覺性：** 考慮到跨平台（Linux/Windows）作業，執行 Shell 指令前必須偵測目前的作業系統與 Shell 環境（如 PowerShell）。
*   **資料隔離：** 嚴格區分「原始素材 (`raw/`)」與「分析結果 (`results/`)」，確保資料流程的可溯源性。

---

## 2. 目錄結構定義 (Directory Structure)

本專案採用嚴格的層級化管理，請勿隨意更動頂級目錄：

| 目錄名稱 | 功能描述 |
| :--- | :--- |
| `working/` | **活動工作區**。每個作品擁有專屬子目錄，存放原始素材 (`raw/`) 與分析暫存檔。 |
| `results/` | **最終資料庫**。存放結構化、具備高度參考價值的最終分析報告。 |
| `classes/` | **專題彙整區**。依屬性（如：背景類型、角色原型）從 `results/` 中提取出的精選集。 |
| `scripts/` | **工具箱**。存放自動化爬蟲、解析腳本與處理工具。 |
| `skills/` | **提示詞庫**。存放可重複使用的 Prompt 片段，輔助 Agent 執行特定分析任務。 |
| `skills_index.md` | **技能索引**。列出所有可用技能與提示詞的中央目錄。 |
| `for_agent.md` | **Agent 執行規範**。定義 Agent 的行為準則、工作流與檢查清單。 |

---

## 3. 標準作業流程 (SOP)

### 作品分析流程
1.  **攝入 (Ingestion)：** 於 `working/<作品名稱>/raw/` 建立資料夾並存入原始素材。
2.  **處理 (Processing)：** 在 `working/<作品名稱>/` 下進行清洗與初步提取，產生中間產物。
3.  **綜合 (Synthesization)：** 將最終分析結果整理為結構化報告，匯出至 `results/<作品名稱>/`。
4.  **彙整 (Aggregation)：** 受命進行跨作品分析時，將結果存入 `classes/` 中的對應主題文件。

### 維護與更新
- 定期更新 `skills_index.md` 以反映 `skills/` 中的最新提示詞。
- 確保 `scripts/` 中的腳本具備跨平台相容性。
