# 技能：概念/設定詞條撰寫 (extract_concepts)

**用途**：根據原始素材與 `entities.md` 名單，撰寫抽象概念與設定詞條，輸出至 `results/<作品>/concepts.md`。重要概念若內容超過約 25 行，拆出 `results/<作品>/concepts/<name>.md`。

---

## 提示詞

根據以下原始素材與 `entities.md` 中的「概念/設定」分類，為《{作品名稱}》撰寫詞條。以繁體中文輸出。

### 適合收錄

- 魔法系統、力量體系（如「源質」、「氣脈」、「以太」）
- 宗教、信仰、教義（不是教團組織——那屬於 factions）
- 創世神話、宇宙觀（一句話的核心命題層級）
- 抽象設定（如「血脈詛咒」、「靈魂契約」、「轉生輪迴」）
- 術語、稱號、職業（若有獨特設定，不是泛泛通用詞）
- 預言、傳說（具名者）

> 注意：具體的事件屬於 `events.md`；具體的物品屬於 `items.md`；具體的組織屬於 `factions.md`。這裡放的是**抽象規則、世界運作原理、信仰系統**。

### 鐵則

- **窮盡名單**：entities 中所有具名概念都要有條目。
- **短條目優先**：3–8 行為默認。核心世界觀骨幹（魔法系統總綱、創世神話）才展開。
- **長條目拆檔**：超過約 25 行拆到 `concepts/<name>.md`。
- **全劇透**：概念的真相（如「神其實是 X」、「魔法系統的禁忌真相」）全寫。
- **不只是定義**：概念條目要寫「這在故事中扮演什麼功能」，不只是字典式釋義。

### 詞條結構（彈性）

```
### {概念名稱}（{原文}）

> {一句話：魔法系統/宗教/世界觀設定/...}

- **意涵**：{在這個世界中代表什麼、怎麼定義}
- **運作方式**：{若是系統性的概念，如何運作、有什麼規則或限制}
- **持有者 / 信仰者**：{誰相信、誰實踐，連結 factions/characters}
- **與其他概念的關係**：{對立、衍生、依存，連結相關概念}
- **故事中的功能**：{如何驅動衝突、推進劇情、塑造世界觀}
- **真相 / 隱情**（可選）：{故事揭露的幕後設定}
```

### 排序

按敘事重要性排序：核心世界觀骨幹在前，邊緣設定在後。

**輸出格式**：Markdown。
