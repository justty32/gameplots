# 技能：物品/技術詞條撰寫 (extract_items)

**用途**：根據原始素材與 `entities.md` 名單，撰寫物品與技術詞條，輸出至 `results/<作品>/items.md`。重要物品若內容超過約 25 行，拆出 `results/<作品>/items/<name>.md`。

---

## 提示詞

根據以下原始素材與 `entities.md` 中的「物品/技術」分類，為《{作品名稱}》撰寫詞條。以繁體中文輸出。

### 適合收錄

- 神器、聖物、傳奇武器（如「神劍」、「創世之鎚」）
- 一般但有名的武器、防具（推動劇情者）
- 魔法物品、卷軸、藥劑（具名者）
- 科技裝置、機械、載具
- 技術系統的具體實作（如「契約獸召喚石」、「龍語符文」）
- 有名稱的載具、船艦、飛行器

> 注意：抽象的魔法系統、技術原理屬於 `concepts.md`；只有具體的物品、有形產出才放這裡。

### 鐵則

- **窮盡名單**：entities 中所有具名物品都要有條目。
- **短條目優先**：3–8 行為默認。推動主線的關鍵物品才展開。
- **長條目拆檔**：超過約 25 行拆到 `items/<name>.md`。
- **全劇透**：物品的真實能力（與表面宣稱不同者）、最終下落、隱藏代價都寫。
- **媒介翻譯**：物品若有重要視覺設計（外觀、發光特效），只在其有敘事意義時轉述。

### 詞條結構（彈性）

```
### {物品/技術名稱}（{原文}）

> {一句話：神器/武器/魔法物品/科技裝置/...}

- **本質**：{這是什麼}
- **能力**：{能做什麼、功能}
- **來源**：{如何誕生、由誰製造、起源故事}
- **持有 / 使用者**：{誰擁有它、誰使用過它，連結 characters}
- **故事中的角色**：{推動了什麼劇情、出現在哪些事件}
- **代價 / 隱情**（可選）：{使用代價、隱藏副作用、真實本質}
- **下落**：{故事結束時在哪、是否被毀}
```

### 排序

按敘事重要性排序。

**輸出格式**：Markdown。
