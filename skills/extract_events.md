# 技能：事件詞條撰寫 (extract_events)

**用途**：根據原始素材與 `entities.md` 名單，撰寫具名事件的詞條，輸出至 `results/<作品>/events.md`。重要事件若內容超過約 25 行，拆出 `results/<作品>/events/<name>.md`。

---

## 提示詞

根據以下原始素材與 `entities.md` 中的「事件」分類，為《{作品名稱}》撰寫事件詞條。以繁體中文輸出。

### 適合收錄的事件

- 戰役、戰爭（如「奧倫攻防戰」）
- 災難、大事件（如「神之黃昏」、「第三次大滅絕」）
- 儀式、典禮（如「神聖加冕儀式」）
- 政變、暗殺、條約等政治轉折
- 預言、徵兆、神秘現象
- 主角團隊的關鍵冒險節點（若有命名）

### 鐵則

- **窮盡名單**：entities 中所有具名事件都要有條目。
- **短條目優先**：3–8 行為默認。對世界觀有長期影響的關鍵事件才展開。
- **長條目拆檔**：超過約 25 行拆到 `events/<name>.md`。
- **全劇透**：事件的隱藏真相、後來才揭露的幕後黑手、長期後果全寫。
- **時間定位**：每個事件都要嘗試標出「何時」（即使是模糊的「神話時代」或「100 年前」）。

### 詞條結構（彈性）

```
### {事件名稱}（{原文}）

> {一句話：戰爭/災難/儀式/...}

- **時間**：{編年位置或時代}
- **地點**：{發生於何處，連結 places 詞條}
- **參與者**：{涉入的人物/陣營，連結對應詞條}
- **經過**：{一段話描述發生了什麼}
- **結果**：{誰贏誰輸、誰死誰活、立即後果}
- **長期影響**：{對世界的持久改變、留下的傷疤}
- **真相 / 隱情**（可選）：{後續故事揭露的幕後}
```

### 排序

按時間順序排列；同時間內按重要性。注意 `timeline.md` 是線性編年表，這裡是詞條深度說明，兩者互補。

**輸出格式**：Markdown。
