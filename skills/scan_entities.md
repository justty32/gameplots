# 技能：全景實體掃描 (scan_entities)

**用途**：在進入詞條撰寫前，先建立作品中所有具名實體的全名單，作為後續 Phase 3 的覆蓋 checklist。輸出至 `working/<作品>/entities.md`。

---

## 提示詞

請通讀以下原始素材，列出《{作品名稱}》中所有「值得單獨記錄一條」的具名實體。以繁體中文輸出。

### 廣度鐵則

- **目標是窮盡而非精選**。配角、地名、術語、儀式、無名事件——只要有名字、有獨立指稱、未來可能被使用者拿來查詢，就必須收錄。
- **重要性的取捨留給 Phase 3**，這一步寧多勿少。漏寫一條，後續詞條庫就缺一條。
- **跨類別的實體**（如「龍族」既是種族也是概念），收在最主要的那一類，並在另一類加 `→ 見 X` 指引。
- **非文字素材**（畫面、配樂、演出橋段）若有專屬名稱或代稱（如某段標誌性 BGM、某個招式名），也要納入；沒有名稱的視覺/音效細節不收。

### 輸出格式

按下列六類整理。每一條只需要：
- **名稱**（含原文 / 常用譯名）
- **一句話**（不超過 30 字）說明這是什麼

```
## 人物 (characters)
- 名稱（原文）— 一句話定位

## 種族/陣營 (factions)
- 名稱（原文）— 一句話定位

## 事件 (events)
- 名稱（原文）— 一句話定位

## 地點 (places)
- 名稱（原文）— 一句話定位

## 物品/技術 (items)
- 名稱（原文）— 一句話定位

## 概念/設定 (concepts)
- 名稱（原文）— 一句話定位
```

### 排序

每類內按敘事重要性大致排序（無法判斷時按出場順序）。

**輸出格式**：Markdown。
