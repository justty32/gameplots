# 技能：速覽合成 (synthesize_synopsis)

**用途**：合成《作品》的 5–10 分鐘速讀，輸出至 `results/<作品>/synopsis.md`。**必須在所有詞條（characters / factions / events / places / items / concepts / timeline）完成後才執行**。

---

## 提示詞

以詞條庫（`characters.md` / `events.md` / `timeline.md` / 等）與原始素材為基礎，合成《{作品名稱}》的速覽。以繁體中文輸出。

### 目標

讓讀者花 **5–10 分鐘**讀完，就能說出：「這部作品在演什麼、味道如何、值不值得記住、之後想查什麼。」

- 比短影片摘要有深度
- 比原作短百倍
- 是「替代體驗」，不是「行銷文宣」

### 結構（彈性，依作品調整段落順序與比重）

#### 1. 鉤子（一段話，約 100 字）

為什麼這部作品值得知道？它在講什麼類型故事中的什麼變奏？是 X 套路 + Y 反轉？是 A 主題的極端推到底？

#### 2. 世界與舞台（約 150–300 字）

不是地理介紹，是「這個世界給人的感覺」——氣氛、奇幻濃度、權力結構、時代感、特殊規則。

#### 3. 開局（約 100–200 字）

故事如何起步？主角群是什麼處境？讀者 / 玩家進入故事時看到的第一個畫面是什麼？

#### 4. 主軸（約 600–1500 字）

完整劇情走向，**全劇透**——含關鍵轉折、主要衝突、結局。

不要逐章流水帳。要挑出構成主敘事的事件鏈：
- 起：是什麼打破了開局的平衡
- 承：主角如何回應、認識了什麼
- 轉：哪個轉折讓故事完全變調（隱藏真相、背叛、巨變）
- 合：最終衝突如何展開、結局是什麼

#### 5. 主要人物速寫（3–5 個人，每人 2–3 句）

挑最關鍵的 3–5 人，每人一段：定位 + 核心動機 + 結局。每人後加 `→ [詳見](characters.md#{錨點})`。

#### 6. 主題與留白（約 100–200 字）

這部作品想說什麼？哪些問題它故意不答？哪些設定刻意保持模糊？

#### 7. 為何記住它（一段話，約 100 字）

這部作品在敘事手法、設定、人物、世界觀上有哪一點是其他作品**沒有的**？這是值得日後查回詞條庫的理由。

### 鐵則

- **全劇透**——使用者要的就是「不看也懂」。結局、死亡、反轉、最終 boss 真實身分，全寫。
- **密度**：避免冗詞與套話。每段都要承載資訊，不要「總而言之」、「值得一提的是」這種填字。
- **不流水帳**：不是「先發生 A 再 B 再 C」，而是「核心衝突如何展開、激化、解決」。
- **媒介翻譯**：對非文字作品，把演出 / 畫面 / 配樂的敘事貢獻翻成文字（「最終戰用了開場片頭曲的小調變奏，呼應主角從希望到墮落的軌跡」）。
- **長度**：1500–3000 字為目標。短到無法承載故事的作品可更短，但低於 1000 字通常代表挖得不夠深。
- **連結回詞條庫**：提到核心人物/事件/地點/概念時，加 markdown 連結到對應詞條，方便讀者跳查。

**輸出格式**：Markdown，使用 `##` 段落標題。
