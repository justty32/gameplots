# 技能：人物詞條撰寫 (extract_characters)

**用途**：根據原始素材與 `entities.md` 名單，撰寫人物詞條，輸出至 `results/<作品>/characters.md`。重要人物若內容超過約 25 行，獨立拆出 `results/<作品>/characters/<name>.md`，主檔留摘要 + 連結。

---

## 提示詞

根據以下原始素材與 `entities.md` 中的「人物」分類，為《{作品名稱}》撰寫人物詞條。以繁體中文輸出。

### 鐵則

- **窮盡名單**：entities 中所有具名人物都必須有條目。寧可短，不可漏。
- **短條目優先**：默認每條目 3–8 行。只有當該人物有完整故事弧線（主角群、核心反派、有重大轉折者）才展開。
- **長條目拆檔**：條目超過約 25 行，獨立拆到 `characters/<name>.md`，主檔保留 1–2 行摘要 + 連結 `→ [詳見](characters/<name>.md)`。
- **全劇透**：結局、死亡、背叛、真實身分、隱藏動機全部寫。
- **媒介翻譯**：人物若有重要視覺/音樂特徵（標誌性配樂、外觀變化、動作設計），轉述其**敘事意義**——「他每次出場都伴隨教堂鐘聲，暗示其偽神身分」是好的；「他穿白袍很帥」不是。

### 詞條結構（彈性，依角色重要性增減）

```
### {名稱}（{原文/英文}）

> {一句話定位：職業/陣營/在故事中的位置}

- **登場與結局**：{何時出場、最終命運}
- **動機**：{他要什麼、為什麼要}
- **行動**：{為了動機做了什麼、造成什麼後果}
- **關係**：{與其他人物的關鍵連結，列點，連結到對方詞條}
- **附註**：{標誌性台詞、視覺/音樂特徵的敘事意義、玩家/讀者爭議點等}
```

並非每條都要填滿所有欄位——空的省略即可。次要角色可能只有「定位 + 登場與結局」就夠了。

### 排序

按敘事重要性排序：主角群 → 主要敵對 → 重要配角 → 次要 → 跑龍套但有名者。同重要性內按出場時間。

**輸出格式**：Markdown。
