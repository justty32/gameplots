# 技能：地點詞條撰寫 (extract_places)

**用途**：根據原始素材與 `entities.md` 名單，撰寫地點詞條，輸出至 `results/<作品>/places.md`。重要地點若內容超過約 25 行，拆出 `results/<作品>/places/<name>.md`。

---

## 提示詞

根據以下原始素材與 `entities.md` 中的「地點」分類，為《{作品名稱}》撰寫地點詞條。以繁體中文輸出。

### 適合收錄的地點

- 國家、王國、帝國
- 城市、都市、要塞
- 地標（神殿、塔、橋、樹）
- 區域（森林、沙漠、海域，若有名稱）
- 異界、其他次元、夢境（若有名稱）
- 重要建築（皇宮、地下城、學院）

### 鐵則

- **窮盡名單**：entities 中所有具名地點都要有條目。
- **短條目優先**：3–8 行為默認。重要地點（首都、終極決戰地、起源之地）才展開。
- **長條目拆檔**：超過約 25 行拆到 `places/<name>.md`。
- **不只地理**：每個地點都要寫**敘事意義**——這個地方在故事中代表什麼？發生過什麼？為什麼值得記？只有地圖座標的地點不該收。
- **全劇透**：被毀滅、被汙染、隱藏的真相、地下祕密都寫。

### 詞條結構（彈性）

```
### {地名}（{原文}）

> {一句話：國家/城市/地標/異界/...}

- **位置**：{在世界中的相對位置、地理特徵}
- **概述**：{這是什麼地方、什麼樣子}
- **居民 / 掌控者**：{誰住在這、誰統治，連結 factions/characters}
- **發生過的事**：{與此地相關的關鍵事件，連結 events}
- **敘事意義**：{這個地點在故事中代表什麼概念或情感}
- **現狀**（可選）：{故事「當下」此地的狀態——繁榮、廢墟、淪陷}
```

### 排序

按敘事重要性排序。同重要性內可按地理區域分組（如「東方諸國」、「北方蠻荒」）。

**輸出格式**：Markdown。
