# 技能：索引建立 (build_index)

**用途**：機械合成 `results/<作品>/index.md`，把所有詞條收集成可瀏覽的索引清單。

---

## 提示詞

掃描 `results/<作品>/` 下的所有詞條檔案——`characters.md` / `factions.md` / `events.md` / `places.md` / `items.md` / `concepts.md`，以及子目錄中的單獨拆檔（如 `characters/<name>.md`）——輸出索引至 `index.md`。以繁體中文輸出。

### 結構

```
# 《{作品名稱}》索引

> 本作小百科的入口。要 5 分鐘讀懂全貌請看 [synopsis.md](synopsis.md)；要按時間順序看事件請看 [timeline.md](timeline.md)。

## 人物
- [{名稱}](characters.md#{錨點}) — {一句話定位}
- [{名稱}](characters/{name}.md) — {一句話定位}（單獨拆檔者）

## 種族 / 陣營
（同上格式）

## 事件
（同上格式）

## 地點
（同上格式）

## 物品 / 技術
（同上格式）

## 概念 / 設定
（同上格式）
```

### 鐵則

- **窮盡**：每一條詞條都要在索引出現一次，不重複、不遺漏。
- **錨點規則**：
  - 主檔內條目連結用 `<檔名>.md#<中文錨點>`（GitHub 風格 slugify，將條目標題轉為錨點）
  - 單獨拆檔者直接連到該檔案，不指向錨點
- **一句話**直接從詞條的「一句話定位」（`>` 區塊）擷取，保持索引乾淨。
- **排序**：每類內按敘事重要性排序，與該詞條檔案內的順序一致。

### 操作流程

1. 讀取每個 `<類>.md` 主檔，列出所有 `### ` 標題與其 `> ` 一句話定位。
2. 列出 `<類>/` 子目錄下的所有單檔。
3. 按上述格式寫入 `index.md`。
4. 自我檢查：每個類別的條目數，與 `entities.md` 對應類別的數量是否一致（若不一致，回報缺漏）。

**輸出格式**：Markdown。
