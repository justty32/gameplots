# 技能：編年表 (build_timeline)

**用途**：產出作品的線性編年表，輸出至 `results/<作品>/timeline.md`。

---

## 提示詞

根據 `events.md` 詞條與原始素材，為《{作品名稱}》建立編年表。以繁體中文輸出。

### 結構

#### 1. 時代分期

把整個世界史劃分為 2–5 個時代，每個時代給一個有畫面感的名稱（如「巨人統治時代」、「諸神黃昏後」），並說明：
- 時代的核心特徵（一句話）
- 如何過渡到下一個時代（通常是某個大事件 / 大災難）

#### 2. 線性事件列

在每個時代下，按時間順序列出該時代的事件。每個事件一行：

```
- {年代或時期} — **{事件名稱}** — {一句話，不超過 30 字} → [詳見](events.md#{錨點})
```

年代未知者用 `[時期不詳]` 或相對時間（如「神話時代後期」、「故事開始前約 50 年」）。

#### 3. 故事「當下」標記

明確標出本作主線發生的時間點（用 `★ 故事主線` 標記），讓讀者知道哪些是背景、哪些是「當下」、哪些是後續。

### 鐵則

- **不重述事件細節**：細節在 `events.md`，這裡只是時間骨架 + 一句話定位 + 連結。
- **窮盡 events.md**：每個 events 詞條都要在編年表上有一行（除非真的時間不明且無法歸類）。
- **時代命名要有畫面**：「上古時代」太懶，「巨人血統紀」具體。

### 範例片段

```
## 第一紀：創世與諸神之爭（神話時代）

> 諸神創造世界與凡人；最終神族內戰結束此紀。

- [起點] — **創世** — 主神以五元素創造大地 → [詳見](events.md#創世)
- [紀末] — **諸神戰爭** — 光暗陣營分裂，光神被弒 → [詳見](events.md#諸神戰爭)

## 第二紀：人類紀元（故事主線發生於此）

...
- [元年]   — **王國奠基** — 第一位人類王登基
- [約 800] — ★ **故事主線開始** — 主角覺醒於奧倫
...
```

**輸出格式**：Markdown。
